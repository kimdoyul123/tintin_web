# 🚀 Edge Function 수동 배포 가이드

## Create Function 버튼이 없을 때

Supabase Dashboard에 "Create Function" 버튼이 없다면, **Supabase CLI**를 사용하세요.

## 방법 1: 배포 스크립트 사용 (가장 쉬움)

1. `배포_스크립트.bat` 파일을 더블클릭하여 실행
2. 브라우저가 열리면 Supabase 계정으로 로그인
3. 자동으로 배포가 진행됩니다!

## 방법 2: 수동으로 명령어 실행

### 1단계: Supabase 로그인

터미널(명령 프롬프트 또는 PowerShell)을 열고:

```bash
npx supabase login
```

브라우저가 열리면 Supabase 계정으로 로그인하세요.

### 2단계: 프로젝트 연결

```bash
npx supabase link --project-ref xdxnvndnzziugiptuvys
```

### 3단계: Edge Function 배포

```bash
npx supabase functions deploy chat-gpt
```

### 4단계: OpenAI API 키 설정

```bash
npx supabase secrets set OPENAI_API_KEY=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
```

### 5단계: Supabase 환경 변수 설정 (선택사항)

Function Calling이 작동하려면:

```bash
npx supabase secrets set SUPABASE_URL=https://xdxnvndnzziugiptuvys.supabase.co
```

Service Role Key는 Supabase Dashboard에서 확인:
1. https://supabase.com/dashboard 접속
2. 프로젝트 선택
3. **Settings** > **API** 클릭
4. **service_role key** 복사
5. 다음 명령어 실행:

```bash
npx supabase secrets set SUPABASE_SERVICE_ROLE_KEY=[복사한_service_role_key]
```

## ✅ 배포 확인

배포가 완료되면:
- 브라우저에서 챗봇을 열고 메시지를 보내보세요
- GPT-4o-mini가 응답하면 성공! 🎉

## 🔍 문제 해결

### "command not found" 또는 "명령을 찾을 수 없습니다"
→ Node.js가 설치되어 있는지 확인하세요. `node --version` 명령어로 확인 가능합니다.

### "Project not found"
→ 프로젝트 ID가 올바른지 확인하세요. Supabase Dashboard에서 프로젝트 설정을 확인하세요.

### "Permission denied" 또는 "인증 실패"
→ `npx supabase login`을 다시 실행하세요.

### 배포는 성공했지만 챗봇이 작동하지 않음
→ Supabase Dashboard > Edge Functions > `chat-gpt` > **Logs** 탭에서 오류 확인
